## 为什么 `range(3)` 是 `0,1,2`？

结合语法规则拆解：

- `range(3)` 属于第一种形式（仅传 `stop=3`），因此默认 `start=0`，`step=1`
- 按「左闭右开」规则，序列需满足：`start ≤ x < stop`（x 是整数，步长 1）
- 代入数值：`0 ≤ x < 3`，满足条件的整数只有 `0, 1, 2`

### lambda匿名函数

![image-20260303115831353](python学习的小细节知识-img/image-20260303115831353.png)

传递计算逻辑，一般只要使用一次就够了

![image-20260303161242163](python学习的小细节知识-img/image-20260303161242163.png)

```py
def func(compute):
    #被计算数据固定为1和2，要计算的逻辑取决于传入的compute函数
    result=compute(1,2)
    print(result)

#调用func4次，完成+-*/
#这些函数仅用一次
func(lambda x,y:x+y)
func(lambda x,y:x-y)
func(lambda x,y:x*y)
func(lambda x,y:x/y)
# 如果想要重复利用，可以将lambda 匿名函数存入变量中即可
# 这个变量就是一个带名字的函数
# chengfa=lambda x,y:x*y
# print(chengfa(3,4))
```

### 异常捕获（异常处理）

==为什么要捕获异常呢？==

`因为代码在运行的时候如果遇到报错则会直接停下来不往后走了，如果此时你捕获了异常，对异常有了预案，程序就会照着你的预案走，不会停下来`







某些bug可以直接跳过，对某些问题有预案

![image-20260304213903634](python学习的小细节知识-img/image-20260304213903634.png)

![image-20260304215644915](python学习的小细节知识-img/image-20260304215644915.png)

![image-20260304215732857](python学习的小细节知识-img/image-20260304215732857.png)

捕获多个异常：

![image-20260304220419317](python学习的小细节知识-img/image-20260304220419317.png)

```py
try:
    可能有异常的代码
except 异常类型 as 变量:
    出现异常的处理代码
except 异常类型 as 变量:
    出现异常的处理代码
......
except 异常类型 as 变量:
    出现异常的处理代码
else:
    没有异常的处理代码
finally:
    有没有异常都会执行的代码
```

==有异常则会进入except分支==

==没有异常，则会进入else分支==

![image-20260304223701913](python学习的小细节知识-img/image-20260304223701913.png)



每个异常类型（类）都是有实体的

```python
except 异常类型 as 变量:      #异常其实也是一个面向对象(类），把对象放到这个变量里面
```



面向对象有个继承的概念，底下的异常都是他的子子孙孙，抓他就都抓住了
